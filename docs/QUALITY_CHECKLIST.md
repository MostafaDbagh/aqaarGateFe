# قائمة التحقق من الجودة — AqaarGate Frontend

## 1. البناء والإطلاق

- [ ] `npm run build` ينتهي بنجاح (بدون أخطاء)
- [ ] `npm run start` يعمل ويخدم الصفحات
- [ ] تعيين متغيرات البيئة في الإنتاج: `NEXT_PUBLIC_SITE_URL`, `NEXT_PUBLIC_API_URL`

## 2. المسارات الحرجة

### نموذج اتصل بنا (Contact)
- [ ] الإرسال بحقول فارغة يعرض رسائل خطأ تحت كل حقل (اسم، هاتف، اهتمام، رسالة)
- [ ] الكتابة في حقل يزيل رسالة الخطأ لذلك الحقل فقط
- [ ] البريد غير الصحيح يعرض خطأ تحت حقل البريد
- [ ] الإرسال الصحيح يظهر مودال نجاح والنموذج يُمسح
- [ ] النسخة العربية تعرض رسائل الخطأ بالعربية

### التحقق من OTP (تسجيل / نسيت كلمة المرور)
- [ ] عند فتح مودال OTP يُحفظ في sessionStorage (استعادة بعد التحديث)
- [ ] بعد إدخال OTP صحيح يُغلق المودال وتظهر مودال النجاح
- [ ] مسح sessionStorage و localStorage عند النجاح يمنع إعادة فتح المودال
- [ ] إغلاق يدوي (X) يمسح التخزين ويغلق المودال

### البحث والـ SEO
- [ ] الصفحة الرئيسية `/en` و `/ar` لها عناوين ووصف وكلمات مفتاحية مناسبة
- [ ] الكلمات المفتاحية العربية (عقار غيت، عقار جيت، عقارات في سوريا، بيوت للبيع) موجودة في metadata و Structured Data
- [ ] hreflang و alternate للغتين يعملان (InternationalSEO)

## 3. الترجمة (i18n)

- [ ] مفاتيح `contact.errors` موجودة في `en.json` و `ar.json`
- [ ] مفاتيح `otpVerification` و `registrationSuccess` موجودة في اللغتين
- [ ] لا توجد صفحات تظهر مفاتيح الترجمة كـ "namespace.key" للمستخدم

## 4. الوصولية (Accessibility)

- [ ] حقول النماذج مرتبطة بـ label (htmlFor / id)
- [ ] رسائل الخطأ لها `role="alert"` أو `aria-describedby` حيث يلزم
- [ ] الحقول التي بها خطأ تستخدم `aria-invalid` عند الحاجة

## 5. الأمان والأداء

- [ ] عدم كشف مفاتيح API أو بيانات حساسة في الكود الأمامي
- [ ] عدم الاعتماد على console.log في مسار الإنتاج (أو إزالتها)
- [ ] منع الإرسال المزدوج في النماذج (مثل Contact: 2 ثوانٍ)

## 6. الاختبار اليدوي الموصى به

| الصفحة / المسار | ما يتم التحقق منه |
|------------------|-------------------|
| `/en/contact` | نموذج اتصل بنا، أخطاء الحقول، إرسال ناجح |
| `/ar/contact` | نفس السلوك مع واجهة عربية |
| `/en` ثم تسجيل | فتح OTP → تحديث الصفحة → إدخال OTP → إغلاق المودال ونجاح |
| `/ar` | الصفحة الرئيسية بالعربية، RTL، روابط تعمل |
| `/en/property-list` | البحث، الفلاتر، عرض القوائم |
| `/ar/property-list` | نفس الوظائف مع واجهة عربية |

## 7. Lint

- [ ] إصلاح أخطاء ESLint الحرجة (react/no-unescaped-entities, set-state-in-effect حيث أمكن)
- [ ] التحقق من أن التحذيرات المتبقية مقبولة أو مُعلّقة بتعليق واضح
